Cowork：把 Claude 从
"一次会话" 变成 "持续在线的同事"

过去 Claude 是"一次对话一段记忆"——你关掉窗口，它就忘了。 Cowork 改变了这件事：同一个持久代理线程跨设备运行， 你在桌面交代任务、外出时手机继续接管、回家电脑里看到结果已经躺好。

07.1 · 一天的样子

真实场景：
给 Claude 一个跨时段任务

09:14
桌面 · 公司

"Claude，把昨天的客户访谈录音整理成一份内部备忘录，对比上周提到的痛点， 最后给销售团队一份要点摘要。"

10:30
手机 · 通勤

地铁里收到推送："已整理初稿，发现有两个新痛点。要不要补一段对应解决方案？" 你在手机上回："好，简短一点。"

11:45
桌面 · 客户公司

打开桌面 Claude——同一个线程，状态显示"已完成"。 看到完整备忘录、销售要点、和你建议的两段补充。

14:20
手机 · 午后

"把这份备忘录改成英文，发给 SF 办公室。" Claude 处理后通过推送通知确认发送。

关键不同：是同一个 Claude，还是同一个线程？

是同一个线程——它保留之前的上下文（用户偏好、文档版本、对话历史）。 不像 ChatGPT 跨设备需要手动切到同一会话，Cowork 的设计假设是 "用户视角下，Claude 一直在"，设备只是访问入口。

"什么时候它通知我？"

三种情况会触发推送：① 任务完成；② Claude 需要确认才能继续（比如要 push 代码、发邮件等不可逆操作）； ③ 出现明确错误。它不会因为"我有个有意思的想法"打扰你—— 这是"尊重用户自主权"在通知策略上的具象。

07.2 · 架构

Cowork 是怎么工作的？

两个核心子系统

Cowork（知识工作）本机执行

跑在你的 Claude Desktop 里——文档处理、邮件起草、表格分析、跨应用自动化。 需要你的电脑保持唤醒、应用打开。Claude 能看到你授权的文件 / 应用 / 浏览器。

Claude Code（开发任务）自动调度

当任务涉及代码——比如 "修个 bug"、"加一个 endpoint"—— Claude 自动 spin up Claude Code 会话；不需要你手动切工具。

持久线程后台

Anthropic 云端保存对话状态、任务进度、待确认的 checkpoints。 你换设备时，从云端拉取最新状态——本地只是渲染端。

推送通道

Claude 移动端注册 push token；任务完成 / 需要确认时由后端发推送。 桌面端打开时显示 inline 通知 + 振动可选。

企业增强（4 月新加）

📊 Analytics API

管理员能拿到每个用户/团队的 engagement & adoption metrics—— 多少任务、平均时长、采纳率、哪些功能没人用。

📡 OpenTelemetry 接入

Claude 的执行轨迹（每个 tool call、每次 model call）能 export 到你自己的 可观测平台——Datadog、Honeycomb、Splunk 都可。这是企业把 Claude 放进生产的关键。

🔐 RBAC + SCIM

把用户分组、对接 IdP 自动同步、按 feature 粒度授权。 单独的"AI 试用"组、"工程师全功能"组、"销售只读"组等划分。

这三件能力让 Cowork 从"个人增强工具"变成 "企业可治理的协作层"—— 也是 4 月最被低估的更新之一。

07.3 · 用之前要知道

Cowork 的限制与注意事项

能做什么

持久线程跨设备记忆
自动判断 Cowork（知识工作）vs Claude Code（开发任务）
桌面应用执行需要电脑唤醒 + 应用打开
推送通知 · 任务完成 / 需确认 / 出错三类
企业版有 Analytics API + OpenTelemetry + RBAC
Pro / Max / Team / Enterprise 均可用，beta 阶段

还做不到（4 月态）

电脑关机时无法继续——它不是云端 agent，需要本地 desktop 执行
不可同时多人协作同一线程——目前是 single-user
第三方应用集成需要单独授权（与 MCP 协议生态相关）

设计哲学映射

持久线程 Helpful 真实工作不是"一次性"的——记忆是协作前提。
桌面端必须打开 Honest 不假装"云端永远在跑"——透明地告诉你执行依赖你的电脑。
推送限三类 用户自主 不打扰是默认；只在必要时通知，避免成为下一个干扰源。
Analytics + OTel Honest "AI 在干嘛"必须能看见——给企业的可审计性。
RBAC + SCIM 用户自主（组织级） 把权限设计延伸到"谁能让 Claude 做什么"。
"需确认才继续" Harmless 不可逆操作即使在 Auto Mode 下也必须显式 ask。

Cowork 与 ChatGPT 的本质区别

ChatGPT Memory 是"记住事实"——名字、偏好、地址。 Cowork 的持久线程是"记住任务"——这件事进行到哪一步、 下一步该等谁、出错时怎么回退。前者是知识库，后者是状态机。
