08.2 · 中间周

v2.1.116 → v2.1.117

v2.1.116
04-20
性能 · 思考可视化

/resume 大型会话快 67%（40MB+ 会话）； VS Code / Cursor / Windsurf 全屏滚动更平滑； 思考微调显示进度提示（"still thinking → thinking more → almost done"）； /config 搜索匹配选项值（搜索 "vim" 找到编辑器模式）； /doctor 在 Claude 响应时可打开，无需等待当前轮次完成。

perf: resume-67%feat: thinking-progressfeat: config-search
v2.1.117
04-22
Subagent fork · 会话总结 · 并发 MCP

外部构建支持 CLAUDE_CODE_FORK_SUBAGENT=1 启用 fork 子代理； Agent frontmatter mcpServers 通过 --agent 加载； /resume 提供"总结大型陈旧会话"的选项——把超长历史摘要后继续； 本地和 claude.ai MCP 服务器并发连接成为默认，启动更快。

feat: subagent-forkfeat: session-summaryperf: parallel-mcp

这两个版本的共同主题：大会话不再是噩梦

67% 的 /resume 提速 + 总结陈旧会话 + 并发 MCP—— 这三件事联合解决了 Claude Code 用户最常抱怨的"打开旧项目慢得想哭"。 看起来是性能修复，本质上是"让长上下文真的可用"—— 与 Opus 4.7 的 200K 上下文相辅相成。
