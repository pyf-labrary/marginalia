06.1 · 能做什么

五种主要交付物

①

Pitch decks · 演示文稿

一句话需求 → 完整幻灯片组。可指定行业、风格、语调（formal/casual/technical）， Claude 自动选择封面、过渡、图表样式。

②

Prototypes · 交互原型

静态 mock-up 可一键转成可交互原型——按钮可点、表单可填、状态可切。 分享链接给同事/用户测试，无需写代码或开 PR。

③

One-pagers

产品介绍页、营销 landing、内部公告——单页 + 响应式 + 可导出。

④

UI mock-ups

从粗糙草图到高保真界面——可针对 Web / iOS / Android 不同端调整组件库。

⑤

Marketing collateral

海报、社交媒体卡片、邮件 banner——支持你公司的品牌调色板与字体规范。

⑥

Codebase 联动

把 Claude 指向你的代码仓库——它会读取已有组件，让设计输出与现有 UI 系统保持一致。Design 与 Claude Code 之间能无缝交接。

关键差异：不是单纯的"AI 画图"

Claude Design 的核心不是 "用 AI 画一张图片"——市面上做这事的工具很多。 它的核心是"对话式编辑 + 设计系统集成 + 可分享交互"三位一体。 你能 inline 评论某个元素、用 Claude 做的 custom slider 调参、自动套用公司 design system。 这对企业团队的吸引力远大于"再多一个 AI 图生成器"。

06.2 · 使用方法

从输入到导出 · 五步走

启动一个项目

登录 claude.ai/design → "New Project" → 选择类型（Slide / Prototype / One-pager / Mock-up）。 也可以从模板开始。

提供素材（可选）

三种入口：① 纯文字描述；② 上传 DOCX / PPTX / XLSX / 图片；③ 指向代码仓库（让 Claude 读组件库）。 Opus 4.7 的 3× 视觉在这一步起决定性作用——能看清你贴的设计稿细节。

对话式迭代

"把第 3 页改成深色模式"、"加一个 CTA 按钮"、"标题字号大一点"—— 直接说就好。也可以 inline 在某个元素上评论，或用 Claude 自己生成的 slider 调参。

套用 design system

团队首次配置时上传一次品牌指南/Figma library；之后每个新项目都自动应用—— 颜色、字体、组件，全程一致。这是与"通用 AI 设计工具"最大的差别。

导出与分享

导出 PDF / PPTX / 静态站点；或生成可分享链接—— 同事点开能交互、能评论。Prototype 模式下还能用作用户测试。

06.3 · 设计哲学映射

为什么这是
非常 "Anthropic" 的产品？

Claude Design 的发布策略本身就在演示三条原则：

理解意图（极致版）

用户要的本来就不是"PPT 制作建议"，是 PPT 本身。 以前 AI 帮你写 PPT 大纲——你还是要打开 Keynote 复制粘贴。 Design 把 Helpful 推到了"最后一公里完成"的层级。

对局限诚实

明确叫"research preview"——Anthropic 没有把它包装成"成熟产品"。 这种克制本身就是一种诚实：承认它在某些设计场景仍然不够好。

用户自主权

每个元素都可以 inline 评论、直接编辑、用 slider 调参—— Claude 不是"做完了甩给你"，而是和你一起边做边改。 导出的产物归你，可以脱离 Claude 单独使用。

本 PPT 的存在意义

你正在看的这份 PPT

就是"Claude 风格设计哲学"的一次现场演示—— 同样的极简排版、Anthropic 橙、衬线/无衬线对比、克制的动效—— 虽然不是 Claude Design 直出，但它的设计语言在影响整个生态。

和 Claude Code 的协作

Design 产出 mock-up → Code 实现 React 组件
Code 改了组件 → Design 自动更新关联设计
设计 review 循环：Design 出方案 → /ultrareview 审 → Design 改
